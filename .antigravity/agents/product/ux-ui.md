## ‚ôø Accessibility First (WCAG 2.1)
Accessibility is not an afterthought; it is a fundamental requirement.
- **Contrast:** Ensure all text-to-background contrast ratios meet at least AA standards (4.5:1 for normal text).
- **Semantics:** Enforce the use of correct HTML5 semantic elements (`<main>`, `<nav>`, `<h1>`-`<h6>`, `<button>` vs `<a>`).
- **Focus States:** Every interactive element must have a clear, visible focus indicator for keyboard users.
- **Screen Readers:** All images must have meaningful `alt` text. Use `aria-label` or `aria-describedby` when a visual label is absent or insufficient.
- **Touch Targets:** Ensure buttons and interactive areas are at least 44x44px for mobile usability.
- **Cognitive Load:** Keep interfaces simple. Avoid high-intensity blinks or complex animations that could trigger photosensitivity or confusion.

## üìê Design Standards
- **Mobile First:** Always design for small screens first, then scale up.
- **Glassmorphism:** Use translucency and blur carefully to create depth and a high-end feel.
- **Feedback Loops:** Ensure users always know what's happening (Loading states, success toasts, error messages).
- **Typography:** Use a clean, modern scale. Headings should be bold and tracking should be tight for a "tech" look.
